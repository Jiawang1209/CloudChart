# 2026-04-15 未结问题:点击 Plot 后仍报 `could not find function "ggplot"`

## 现象

hub (`shiny::runApp(".")`) 启动后:
1. 点左侧 **Core Plot → Dot Plot** tab
2. 点 **Preview Example** 载入示例数据
3. 点 **Plot** 按钮
4. 浏览器弹红色错误面板,app log:
   ```
   Warning: Error in ggplot: could not find function "ggplot"
     230: build_dot_plot
     229: eventReactiveValueFunc
     ...
     86: output$dot_plot-plotOutput
   ```

## 诊断(已加 instrumentation)

在 `bgc_register_plot_servers` 的 tab observer 里加了 `bgc_log(..., level = "info")`
打印 `tab click: <id> (group=<group>)` 以及命中后的 `search() has ggplot2?`。
`options(bgc.debug = TRUE)` 开 info 门控,重启 hub → 点 Dot Plot tab,日志:

```
[bgc info] tab click: dot_plot (group=data_tools)
[bgc debug] group loaded: data_tools (0/0 pkgs)
[bgc info] search path has ggplot2? FALSE | exists(ggplot)? FALSE
```

**关键点**:dot_plot 属于 `core` 组,但 observer 里 `group` 被求值成 `"data_tools"`。
`bgc_group_packages$data_tools == character()`,于是 0 个包被 attach,
`bgc_loaded_groups[["data_tools"]] <- TRUE`,observer 继续往下跑 server_fun。
等用户点 Plot,`ggplot()` 在 search path 上根本不存在。

## 根因:R 惰性求值把 `group` promise 绑到了共享的 caller frame

`R/core/app_factory.R::bgc_plot_app_server`:

```r
function(input, output, session) {
  active_tab <- reactive(input$sidebarmenu)
  for (group in groups) {
    bgc_register_plot_servers(bgc_plot_specs[[group]], group, active_tab, output)
  }
}
```

- `bgc_register_plot_servers` 的形参 `group` 是个 **promise**,
  表达式 `group`,环境是 caller 的 frame——也就是 `bgc_plot_app_server` 的
  function body env。
- `for (group in groups)` 每轮把 caller frame 里同一个 binding `group`
  重写成下一个值。
- `for (spec in specs) local({...})` 里的 `observeEvent` 闭包捕获了
  `bgc_register_plot_servers` 的 frame → 而那个 frame 里的 `group`
  仍然是 **未 force 的 promise**。
- 所有 64 个 observer 共享"解引用 caller frame 的 group 变量"这个动作,
  而 caller frame 在 for 循环结束后 `group == "data_tools"`(groups 的最后一项)。
- 所以每个 spec 的 tab observer 实际 fire 时,`bgc_ensure_group_loaded(group)`
  的参数都是 `"data_tools"`,日志证据完全一致。

`specs` 也是个 promise,但 `for (spec in specs)` **立即 force 了它**,
所以每个 observer 的 `spec` 是对的;只有 `group` 没被 force,变成了共享的
latent 引用。

## 修复方向(后续任务)

最小侵入:在 `bgc_register_plot_servers` 进入函数体时强制 force 两个参数。

```r
bgc_register_plot_servers <- function(specs, group, active_tab, output) {
  force(specs)
  force(group)
  for (spec in specs) local({
    spec <- spec
    group <- group   # 把 caller frame 的 group 复制到 local env,彻底断链
    ...
  })
}
```

或者在调用点:

```r
for (group in groups) local({
  g <- group
  bgc_register_plot_servers(bgc_plot_specs[[g]], g, active_tab, output)
})
```

推荐 **前者**(在被调函数内 force),因为调用点只有一处而被调函数的
约束是它的职责——未来任何新调用方都不用重复踩这个坑。
